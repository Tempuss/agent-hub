# 통합 워크플로우 템플릿 (Integrated Research + Thinking Workflow)

실제 의사결정에 바로 적용할 수 있는 단계별 템플릿

---

## 📋 템플릿 사용 가이드

각 섹션을 작성하면서 진행합니다. 시간이 없으면 **필수 항목(*)만** 작성해도 됩니다.

---

## Phase 1: 문제 정의 (30-60분)

### 문제 설명
```
현재 상황:
[구체적 현황 작성]

증상 vs 근본 원인:
[이것이 증상인지 근본 원인인지 판단]

비즈니스 영향:
- 재무적 영향: [손실액, 수익 기회]
- 전략적 영향: [사업 영향]
- 운영 영향: [작업량, 효율성]
```

### 성공 기준 *
```
이 문제가 해결되었다는 증거:
1. [측정 가능한 지표]
2. [측정 가능한 지표]
3. [측정 가능한 지표]

현재: [현재값]
목표: [목표값]
기간: [달성 기간]
```

### 스테이크홀더
```
누가 이 문제로 영향받나?
- [역할]: [영향도]

누가 결정권을 가지나?
- [결정권자]

누가 실행하나?
- [실행자]
```

## Phase 2: 사고법 선택 및 적용 (20-60분)

### 사고법 선택 이유
```
문제 유형: [근본원인/혁신/전략/개선/의사결정]
선택한 사고법: [구체적 방법명]
선택 이유: [왜 이 방법이 가장 적합한가]
```

### 사고법 적용 결과

#### 분석 방법: [선택한 사고법]

**주요 발견**:
```
1. [핵심 발견 1]
2. [핵심 발견 2]
3. [핵심 발견 3]
```

**상세 분석**:
```
[선택한 사고법에 따른 상세 분석 기록]

예시 (5 Why인 경우):
Why 1: [첫 번째 원인]
Why 2: [두 번째 원인]
Why 3: [세 번째 원인]
Why 4: [네 번째 원인]
Why 5: [근본 원인]

예시 (Pareto인 경우):
상위 20% = [상위 항목들]
이것이 전체의 [X]% 설명
```

### 초기 신뢰도 평가
```
방법 신뢰도: [%]
  근거: [왜 이 점수인가]

현재 정보 신뢰도: [%]
  사용한 출처: [내부/외부, Tier]

초기 신뢰도: [%]
  = 방법 신뢰도 × 현재 정보
  = [X]% × [Y]% = [Z]%

결론: 현재 신뢰도로 결정 가능한가?
[ ] 예 (80% 이상) → Phase 5로
[ ] 아니요 (60% 미만) → Phase 3로
```

### 연구 필요성 판단

2. [가정 2]
3. [가정 3]

우선순위 (영향도 × 불확실성):
1. [가장 중요한 것]
2. [다음 중요한 것]
```

### 연구 계획 *
```

3. [조사할 항목] → 출처: [Tier]

출처 찾는 방법:
- 검색어: [구체적 검색어]
- 플랫폼: [어디서 찾을 것인가]
- 대안: [만약 찾지 못하면]
```

### 연구 시간 예산
```
이상적: [시간]

## Phase 5: 통합 분석 및 신뢰도 재계산

### 사고법 분석 + 연구 증거 통합

```
초기 사고법 분석:
[Phase 2에서의 결론]

연구로 얻은 증거:
[Phase 4에서의 주요 발견]

통합 결론:
[사고법 + 증거를 함께 고려한 최종 판단]

변화된 점:
[초기 분석과 비교해서 어떤 부분이 달라졌나]
```

### 신뢰도 최종 계산 *

```
방법 신뢰도: [%]
  사용한 방법: [사고법명]
  방법 신뢰도 근거: [왜 이 점수인가]

증거 신뢰도: [%]
  사용한 출처들:
  - Tier 1: [개수] (신뢰도 90%)
  - Tier 2: [개수] (신뢰도 70%)
  - Tier 3: [개수] (신뢰도 50%)
  - Tier 4: [개수] (신뢰도 30%)

  평균 증거 신뢰도 = ([T1개수]×0.9 + [T2개수]×0.7 + [T3개수]×0.5 + [T4개수]×0.3) / 총개수
                  = [%]

상황 적합도: [%]
  시간: [충분/제약]
  리소스: [충분/제약]
  문제-방법 적합도: [적합도]

최종 신뢰도 점수: [%]
  = 방법 신뢰도 × 증거 신뢰도 × 상황 적합도
  = [X]% × [Y]% × [Z]% = [최종]%
```

### 신뢰도 해석
```
신뢰도 80% 이상?
[ ] 예 → 자신감 있게 의사결정 실행 가능

신뢰도 60-79%?
[ ] 예 → MVP/파일럿 규모로 테스트 후 확대

신뢰도 60% 미만?
[ ] 예 → 추가 연구 권장 또는 전문가 상담
```

## 📝 문서 작성 팁

- **명확함**: 나중에 자신이 읽을 때 이해할 수 있게 작성
- **근거**: 판단을 내린 이유를 항상 기록
- **출처**: 사용한 정보의 출처를 명확히
- **수정 허용**: 진행하면서 새로운 정보가 나오면 수정해도 됨
